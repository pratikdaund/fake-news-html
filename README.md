# Veritas — Fake News Detection System

> M.Tech academic project · NLP + Deep Learning · Flask + HTML/CSS/JS · Render

A web application that classifies news articles as **REAL** or **FAKE** using:
- **TF-IDF + Logistic Regression** — classical NLP baseline (~99% on ISOT)
- **Bidirectional LSTM** — deep learning model with trainable embeddings

Built with a Flask backend and a vanilla HTML/CSS/JavaScript frontend — no React, no Streamlit, no heavy frameworks.

## 🚀 Quick Start

### 1. Set up environment

```bash
cd fake-news-detector
python -m venv venv

# Activate:
# Windows: venv\Scripts\activate
# macOS/Linux: source venv/bin/activate

pip install --upgrade pip
pip install -r requirements.txt
```

### 2. Download the dataset

Get the ISOT Fake News Dataset from Kaggle:
https://www.kaggle.com/datasets/clmentbisaillon/fake-and-real-news-dataset

Place `Fake.csv` and `True.csv` inside the `data/` folder.

### 3. Train the models

```bash
# Full training (both models, 5–15 min)
python train.py

# Logistic Regression only (faster, lighter for deployment)
python train.py --skip-lstm

# Quick smoke test
python train.py --sample 5000
```

### 4. Run the app locally

```bash
python app.py
```

Open http://localhost:5000 in your browser.

### 5. Deploy to Render

See deployment section below.

## 🌐 Deploying to Render (Free Tier)

### Step-by-step

1. **Commit trained models to Git.**
   ```bash
   git add models/logreg_model.pkl models/tfidf_vectorizer.pkl
   git add models/lstm_model.keras models/lstm_tokenizer.pkl   # optional
   git commit -m "Add trained models"
   ```

2. **Push to GitHub.** Your repo must be public for free-tier Render, or connect your GitHub account.

3. **Sign up at [render.com](https://render.com)** with your GitHub account.

4. **New + → Blueprint** — select your repo. Render reads `render.yaml` and configures the service automatically. Click **Apply**.

   *Manual alternative:* New + → Web Service, then:
   - **Runtime:** Python 3
   - **Build command:** `pip install -r requirements.txt`
   - **Start command:** `gunicorn --bind 0.0.0.0:$PORT --workers 1 --timeout 120 --preload app:app`
   - **Plan:** Free

5. **Wait ~10 minutes** for first build. Your URL will be `https://fake-news-detector-xxxx.onrender.com`.

### Free-tier limitations

- **Sleeps after 15 min** of inactivity. Cold start ~30-60 seconds.
- **512 MB RAM** — TensorFlow + LSTM fits, but tightly.
- **First visit after idle** may time out. Ping the URL 30 min before any demo.

### If you run out of memory

```bash
# Drop TensorFlow to save RAM
mv requirements.txt requirements-full.txt
mv requirements-light.txt requirements.txt
rm models/lstm_model.keras models/lstm_tokenizer.pkl
git add -A
git commit -m "Lightweight deployment - no LSTM"
git push
```

The app auto-detects missing LSTM files and falls back to Logistic Regression only.

## 🧪 How the System Works

### Request flow

```
User pastes article
        ↓
JavaScript POST /api/predict {text, model}
        ↓
Flask app.py
        ↓
clean_text()  ← lowercase, strip URLs, lemmatise, drop stopwords
        ↓
TF-IDF vectorise  OR  tokenise + pad (for LSTM)
        ↓
Model.predict_proba()  →  real_prob
        ↓
JSON response  →  JavaScript updates DOM
```

### NLP pipeline (in `clean_text`)

1. Lowercase conversion
2. URL removal (`http://...`, `www...`)
3. HTML tag removal
4. Non-alphabetic character stripping
5. Stopword removal (NLTK English list)
6. Lemmatisation (WordNet)
7. Short-token filtering (< 3 chars)

### Deep Learning model (Bi-LSTM)

```
Embedding (64-dim, vocab=10,000)
    ↓
SpatialDropout1D (0.3)
    ↓
Bidirectional LSTM (48 units × 2)
    ↓
Dense (24, ReLU) → Dropout (0.4)
    ↓
Dense (1, Sigmoid)
```

Loss: binary crossentropy · Optimiser: Adam · Early stopping on val_loss.

## 🔬 Expected Accuracy

On ISOT test split (20%):

| Model | Accuracy |
|---|---|
| Logistic Regression | ~99.3% |
| Bidirectional LSTM | ~99.5% |

**⚠️ The caveat that matters.** These numbers reflect the ISOT dataset's structural bias — real news is Reuters-only, fake news is from ~200 dubious sites. Your model learns Reuters stylistics, not truth. Cross-domain accuracy typically drops to 60-75%. **State this honestly in your thesis limitations section.**

## 🛠 Troubleshooting

**`FileNotFoundError: Fake.csv`** — Place dataset CSVs in `data/` folder.

**`No trained models found`** — Run `python train.py` first.

**Port 5000 already in use** — Another app is using it. Change in `app.py` or kill the other process.

**NLTK download fails** — Run manually:
```bash
python -c "import nltk; nltk.download('stopwords'); nltk.download('wordnet'); nltk.download('omw-1.4')"
```

**Render build fails: "No space left on device"** — Use `requirements-light.txt`. TensorFlow wheel is ~500MB.

**Render "Out of memory"** — See the lightweight deployment steps above.

**Cold start timeout on Render free tier** — Known issue. Render takes 30-60s to wake the service. Not a bug.

**Flask shows "Method Not Allowed"** — You hit `/api/predict` with GET. It's POST-only.
